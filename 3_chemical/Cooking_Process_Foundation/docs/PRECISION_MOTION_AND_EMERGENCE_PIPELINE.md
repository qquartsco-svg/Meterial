# 정밀 공정 · 동선 · 경험 → 창발 파이프라인 (설계 정렬)

> **한국어 (정본).** English: [PRECISION_MOTION_AND_EMERGENCE_PIPELINE_EN.md](PRECISION_MOTION_AND_EMERGENCE_PIPELINE_EN.md)

## 1. 사용자가 말한 흐름 — 맞는 이해인가?

**맞다.** 한 줄로는 다음과 같다.

1. **레시피 입력** → 공정이 **동역학(시간·상태·게이트)** 으로 정렬되어 실행되고, **결과(조리 산출)** 가 나온다.  
2. 반복·로그·보정이 쌓이면 **습관·스킬**(개체/로봇 단위 파라미터)이 된다.  
3. **정보·자극**이 지속되면 SIK 등 저수준 루프와, 느린 축에서 **아이디어·형상 후보**(이데아 엔진 등)로 **창발**이 가능하다.

다만 **“오차 없음”** 은 제품 문장으로는 **절대값**이 아니라, **계량·동작·동선 각각에 대한 허용오차·검증·감사**로 구현해야 한다. 아래에서 레이어별로 나눈다.

---

## 2. 제과제빵사 수준의 “정확한 계량·작업” — 이 시스템에서의 위치

| 구분 | 의미 | 현재 CPF (`Cooking_Process_Foundation`) | 다음 계약(권장) |
|------|------|----------------------------------------|------------------|
| **레시피 논리** | 단계 순서·시간·온도·갈변 게이트 | ✅ `RecipeFlow` / `FlowEngine` | 스텝별 **목표 질량·부피·점도 프록시** 필드 |
| **계량 정밀** | g/ml/°C 단위와 불확실도 σ | ⚠️ 열·갈변 **프록시** 수준 | 별도 **`MetrologyStep`** 또는 스텝 `extras` + 저울/유량계 어댑터 |
| **오차 없이 진행** | 실패 시 분기·재시도·폐기 | ⚠️ 게이트는 있음, **계량 합격/불합격** 자동 분기는 확장 필요 | `next_on_branch` + 센서 임계 + **감사 로그** |

**정리**: “레시피 입력 → 동역학으로 흐름 정렬”은 **맞는 방향**이고, **제과제빵 급 정밀**은 **같은 레이어에 우겨 넣지 말고** **계량·검침 서브계약**으로 얹는 것이 레이어가 안 꼬인다.

---

## 3. 실제 로봇 + 동선 체크 + “자율주행”에 가까운 이동

레시피만으로 **관절 궤적·AMR 경로**가 자동 완성되지는 않는다. 대신 **역할 분리**가 맞다.

| 층 | 역할 | 00_BRAIN 쪽 후보 |
|----|------|------------------|
| **조리 공정** | 무엇을 언제 얼마나(의도) | CPF + `ActuatorIntent` |
| **매니퓰레이터 동선** | 충돌 없는 궤적, 작업공간 | 별도 **모션/조작 스택** 또는 기존 자율·플랫폼 엔진 브리지 |
| **이동(자율주행)** | 장애물·경로·속도 | `Autonomy_Runtime_Stack`, `Vehicle_Platform_Foundation`, `4WD`, TAM 등 **이동 전용** |
| **합성** | 공정 일시정지·ESTOP·허가 | 이미 있는 **`Arbiter`** + 향후 **“motion_ok” 게이트** bool |

**“레시피 입력되면 동역학으로 정렬”** = **공정 상태기계**.  
**“동선 체크”** = **기하·제약 검증기** — CPF 출력과 **병렬로** 돌고, 둘 다 만족할 때만 실행 허가하는 형태가 상용에 가깝다.

## 4. 경험 → 습관 → 정보 → 자극 → 아이디어

| 단계 | 메커니즘 (개념) | 저장소/엔진 후보 |
|------|----------------|------------------|
| 경험 | 틱 로그·성공/실패 | 세션 DB, L4 롤업 |
| 습관·스킬 | 파라미터·허용오차 수렴 | MemoryPhase / MAK, 개인 에이전트 프로파일 |
| 지속 자극 | SIK·센서 | `_staging/Sensory_Input_Kernel` |
| 창발 | 느린 통찰 큐 | `ionia.idea` (사람 승인 후 풀링 권장) |

이 흐름은 [ROBOT_SIK_IDEA_INTEGRATION.md](ROBOT_SIK_IDEA_INTEGRATION.md) 및 [INTELLIGENCE_AND_PERSONAL_AGENT_ROADMAP.md](../../design_workspace/INTELLIGENCE_AND_PERSONAL_AGENT_ROADMAP.md) 와 같은 그림이다.

## 5. 한 줄 결론

- **“레시피 → 동역학 정렬 → 조리 결과 → 경험 축적 → 습관 → 자극 → 창발”** 이라는 **서사·아키텍처 방향은 맞다.**  
- **제과제빵 급 오차 최소화**는 **계량·검침 계약 + 하드웨어 캘리브레이션**을 **CPF 위에 별도로** 두어야 레이어가 유지된다.  
- **동선·자율 이동**은 **공정 엔진과 분리된 모션/AMR 층** + **게이트 합성**으로 가는 것이 맞다.
